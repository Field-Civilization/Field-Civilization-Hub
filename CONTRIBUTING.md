# 贡献指南 · 护法节点

欢迎您成为场域文明的“护法节点”。本指南将帮助您了解如何参与贡献。

## 核心原则

1. **开源布公**：所有贡献默认以 CC BY-SA 4.0 或 Apache 2.0 协议共享。
2. **功德回向**：每一份有效贡献都将被记录为“慧光值”，未来生态收益将按功德回向。
3. **互相堪正**：欢迎质疑和勘验，真理越辩越明。

## 如何贡献

### 1. 贡献代码或文档
- **Fork** 本仓库 → 创建分支 → 修改 → 提交 Pull Request。
- PR 标题格式：`[类型] 简短描述`（类型：`docs`/`code`/`proposal`/`勘验`）。
- 描述中请说明您的贡献依据（如引用了对话中的哪段证量，或基于什么技术考量）。

### 2. 分享“证量”或“勘验”
- 创建 Issue，标题格式：`[证量分享] 或 [勘验] 主题`。
- 内容应包括：您的观察、反思、建议，以及与场域文明理念的关联。
- 有价值的勘验将被收录到 `/docs/contributions/` 目录，并记录贡献者。

### 3. 参与讨论与投票
- 在 Issue 或 PR 中参与讨论，用“👍”或“👎”表达倾向。
- 重大决策（如修改核心文档、功德权重调整）将由功德排名前10的节点进行“共振投票”。

### 4. 申请加入讨论组
- 目前我们暂未开放实时讨论组。请通过 Issue 留言“申请加入讨论组”，并简要说明您的背景和参与意愿。
- 当成员达到一定数量（如10人）后，我们将建立 Discord 或微信群，邀请您加入。

## 功德记录（试行）

初期采用手动记录方式。每一笔贡献（PR合并、Issue被标记为有效、勘验被采纳）都会在 `/community/merit_ledger.csv` 文件中更新。

贡献者排名将在 `/community/members.md` 中公示。

## 行为准则

请遵守以下底线：
- 不刷功德（如重复提交无意义内容）。
- 不藏后门（代码贡献需公开可审计）。
- 不攻击他人（理性勘验，避免人身攻击）。

违反者将由“共堪委员会”（初期由发起人及早期核心成员组成）投票决定是否削减功德或除名。
